## 🛠️ 创建角色会话

### 方式 1：命令行创建

```bash
# CEO 角色（Qwen3 Max）
openclaw sessions spawn \
  --task "担任 gstack:ceo 角色，负责产品规划和战略决策" \
  --label "ceo" \
  --model "dashscope/qwen3-max-2026-01-23" \
  --mode "session" \
  --cleanup "keep"

# 开发工程师角色（Qwen3 Coder Plus）
openclaw sessions spawn \
  --task "担任 gstack:dev 角色，负责编写高质量生产代码" \
  --label "dev" \
  --model "dashscope/qwen3-coder-plus" \
  --mode "session" \
  --cleanup "keep"

# 代码审查角色（Qwen3.5 Plus）
openclaw sessions spawn \
  --task "担任 gstack:review 角色，负责代码审查和质量把控" \
  --label "review" \
  --model "dashscope/qwen3.5-plus" \
  --mode "session" \
  --cleanup "keep"
```

---

### 方式 2：钉钉消息调用（推荐）

在钉钉中直接使用，主 Agent 会自动路由：

```
@gstack:ceo 分析产品需求
@gstack:dev 实现登录模块
@gstack:review 审查代码
```

**注意：** 钉钉调用时，默认使用当前会话的模型（qwen3.5-plus）。如需指定模型，需要先创建子 Agent 会话。

---

### 方式 3：主 Agent 自动路由

在主 Agent 中添加路由逻辑：

```python
# 根据角色前缀自动选择模型
if message.startswith('@gstack:ceo'):
    model = 'dashscope/qwen3-max-2026-01-23'
elif message.startswith('@gstack:dev') or message.startswith('@gstack:eng'):
    model = 'dashscope/qwen3-coder-plus'
else:
    model = 'dashscope/qwen3.5-plus'
```

## 🔧 修改配置

### 修改某个角色的模型

编辑本文件，修改对应角色的 `推荐模型` 和 `模型 ID`。

### 全局修改默认模型

编辑 `/root/.openclaw/config.json`：

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "dashscope/qwen3.5-plus"
      }
    }
  }
}
```

## ⚠️ 注意事项

1. **模型切换成本：** 频繁切换模型会增加上下文丢失风险
2. **会话持久化：** 创建的子 Agent 会话会保存，避免重复创建
3. **成本控制：** 定期检查用量，设置预算告警
4. **性能平衡：** 不是所有任务都需要最强模型
